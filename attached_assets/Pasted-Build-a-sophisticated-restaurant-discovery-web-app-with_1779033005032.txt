Build a sophisticated restaurant discovery web app with a map interface and an AI chatbot assistant.

The app should help users find restaurants, compare options, navigate to a restaurant, and decide what to order based on real menu items, user preferences, and public review consensus.

App concept:
The screen should be split into two main areas:
1. A large interactive map on the left
2. A chatbot / recommendation assistant panel on the right

The app should feel premium, modern, clean, and suitable for fine-dining and casual restaurant discovery.

Core user flow:
1. User opens the app
2. The app asks for the user’s location or lets them search a city/area manually
3. Restaurants appear on the map as pins
4. User can filter by cuisine, price level, rating, distance, opening status, dietary preference, and dining style
5. User can click a restaurant pin to see a summary card
6. The chatbot helps the user choose where to go
7. After selecting a restaurant, the chatbot recommends what to order
8. The app can generate a short “dining plan” with restaurant, dishes, navigation link, and booking/contact details if available

Main layout:
- Left side: interactive map
- Right side: AI chatbot panel
- Top bar: search bar, location selector, cuisine filter, rating filter
- Bottom or floating restaurant cards when restaurants are selected
- Mobile version should stack the chatbot and map cleanly

Use:
- React frontend
- Node.js / Express backend if needed
- Supabase for database
- Google Maps API or Mapbox for map interface
- Google Places API or another restaurant data API for restaurant search
- OpenAI API or Anthropic Claude API for chatbot responses
- Environment variables for all API keys
- Clean reusable component structure

Important:
Do not hardcode API keys.
Use environment variables for:
GOOGLE_MAPS_API_KEY
GOOGLE_PLACES_API_KEY
OPENAI_API_KEY
SUPABASE_URL
SUPABASE_ANON_KEY

Map features:
- Show user location if permission is granted
- Allow manual search by city, district, or landmark
- Show restaurant pins on the map
- Clicking a pin opens a restaurant preview
- Restaurant preview should include:
  - Restaurant name
  - Rating
  - Number of reviews if available
  - Cuisine/type
  - Price level if available
  - Distance from user
  - Open/closed status if available
  - Short summary
  - Button: “Ask AI about this place”
  - Button: “Get directions”
  - Button: “Add to shortlist”

Chatbot features:
The chatbot should act as a credible restaurant recommendation analyst, not a generic assistant.

The chatbot should ask clarifying questions before recommending:
- Number of people
- Dietary restrictions
- Seafood / meat / vegetarian preference
- Sushi/raw vs cooked preference
- Light vs indulgent meal
- Sharing plates vs individual meals
- Budget sensitivity only if the user asks or implies it
- Occasion: casual, business, date, family, fine dining, celebration

The chatbot should help with:
- Choosing between restaurants
- Explaining why one restaurant may be better than another
- Recommending what to order
- Building a balanced order for the group
- Avoiding dishes that conflict with user preferences
- Explaining whether the evidence for a dish is strong, moderate, or weak
- Giving navigation guidance
- Creating a final dining plan

Dish recommendation rules:
The chatbot must not hallucinate menu items.
If the menu is unavailable, it must say so clearly and recommend based only on available evidence.
If menu data is available, recommendations must be based on:
- Actual menu items
- Repeated customer review mentions
- Google Reviews / Tripadvisor / Yelp / public review consensus where available
- Restaurant signature dishes where available
- Critic/blog mentions where available

The chatbot must distinguish:
- Signature dishes
- Customer favorites
- Critic favorites
- Frequently recommended dishes
- Weak-evidence dishes

When recommending dishes, structure the answer as:
1. Starters
2. Salads
3. Sushi / Raw, only if relevant
4. Main Courses
5. Desserts
6. Final suggested order

For each dish include:
- Dish name
- RAW or COOKED
- Score from 1 to 10
- Short factual description
- Evidence level: strong, moderate, or weak
- Very short reason for recommending it

Do not show dish prices unless the user specifically asks.
If the user asks for cost, provide only an approximate total estimate and clearly say it is approximate.

Restaurant comparison features:
When multiple restaurants are selected, the chatbot should compare them using:
- Cuisine fit
- Review strength
- Distance
- Opening status
- Dining occasion fit
- Menu fit based on user preference
- Risk level, such as limited menu evidence or weak reviews
